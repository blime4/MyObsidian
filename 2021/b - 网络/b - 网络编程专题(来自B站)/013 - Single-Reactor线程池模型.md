# 013 - Single-Reactor线程池模型

![[Pasted image 20211228193318.png]]


划分IO线程和工作线程
+ IO线程处理IO操作
+ 工作线程处理业务逻辑

+ 改进思路
	+ 引入了线程池，用来专门处理业务逻辑操作，提升I/0响应速度口
+ 缺陷
	+ 虽然在引入线程池后，IO的响应速度提升了不少，但在管理百万级连接、高并发大数据量时，单个 Reactor线程仍然会效率比较底下。


由于Single-Reactor线程池模型，使用单个reactor，面对高并发的场景，还是效率不行
